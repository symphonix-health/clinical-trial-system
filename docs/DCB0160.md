# CTMS DCB0160 Deployer Responsibilities

## 1. Purpose

This document sets out the responsibilities of organisations deploying CTMS in a live environment, as required by NHS Digital Clinical Safety Standard DCB0160.

## 2. Organisational responsibilities

### 2.1 Governance

- Appoint a clinical safety officer (CSO) accountable for the CTMS safety case.
- Maintain a risk management file aligned with `docs/SAFETY_CASE.md`.
- Review and approve configuration changes that affect subject safety or data integrity.

### 2.2 Training

- Train sponsor managers, PIs, CRCs, CRAs, safety officers, site pharmacists, and agent owners on CTMS workflows.
- Document training completion before granting system access.
- Provide refresher training when protocol amendments or system updates change workflows.

### 2.3 Incident management

- Define procedures for reporting and investigating safety-related incidents.
- Escalate incidents involving SUSAR deadline misses, wrong-subject enrolment, or agent autonomy escalation within 24 hours.
- Record corrective and preventive actions (CAPA) and link them to CTMS queries.

## 3. Technical responsibilities

### 3.1 Hosting

- Deploy CTMS on infrastructure that meets organisational security and availability policies.
- Use the Docker Compose stack in `docker-compose.yml` or an equivalent orchestrated environment.
- Maintain database backups with a recovery point objective consistent with trial requirements.

### 3.2 Identity and access management

- Configure the OAuth 2.0 / Keycloak realm with the required roles and scopes.
- Enforce role hierarchy to prevent unblinding leaks.
- Disable accounts promptly on staff departure or role change.

### 3.3 Integration

- Register CTMS as `clinical_trial_system` in BulletTrain's connector manifest only after standalone verification is complete.
- Verify HMAC webhook secrets for inbound and outbound integrations.
- Monitor integration health and queue failed deliveries for retry.

### 3.4 Monitoring

- Collect OpenTelemetry traces for cross-system calls.
- Monitor Prometheus metrics for enrolment rates, AE reporting latency, protocol deviation counts, and eTMF completeness.
- Configure alerts for SUSAR deadlines, ethics expiry, and failed webhook deliveries.

## 4. Clinical responsibilities

- Ensure protocols, amendments, and consent versions are loaded correctly before site activation.
- Verify that site activation checklists are complete and evidence is attached.
- Review safety dashboards daily during active recruitment.
- Investigate and resolve protocol deviations and queries within defined timeframes.

## 5. Validation and verification

- Run the full pytest suite (≥ 75 tests, ≥ 85% coverage) before each deployment.
- Run Playwright E2E tests in both headed and headless modes.
- Run axe-core accessibility scans and resolve serious violations.
- Execute CAID scenario matrices for journeys J20–J35.

## 6. Change control

- All configuration, code, and documentation changes are reviewed and approved.
- Changes that affect safety requirements trigger re-validation of the safety case.
- Release notes document known issues, residual risks, and required actions for deployers.
